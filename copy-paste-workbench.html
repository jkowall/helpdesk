Freshdesk Theme + Deploy Workbench

This page is for two different jobs: full Freshdesk theme updates and smaller deploy-fragment updates. It keeps those paths separate, lets you copy any block with one click, and prefers rendered `dist/` files when they exist so your local credentials stay out of git.

Open Theme Preview Open Widget Preview
Best way to view this Serve the folder over HTTP so file loading works: `python3 -m http.server 8123` and open `http://127.0.0.1:8123/`.
Full theme updates Use the Theme Editor section below when you want to paste complete Freshdesk sections like Head, Header, Footer, Layout, Portal Home, or CSS.
Deploy fragments Use the Deploy Fragments section only for the smaller admin copy/paste snippets such as widget HTML, CSS, and footer script blocks.
Rendered vs source If `dist/` exists, this workbench will load that first. Otherwise it falls back to tracked source files with placeholders.
Secret safety `.env` and `dist/` stay ignored by git. Use `python3 scripts/render_templates.py` when you need local rendered values before copying.

Full Theme Copy/Paste

Use this when you are updating the actual Freshdesk theme editor section by section.

  1. Open Freshdesk theme editing for the portal.
  2. Work straight down the Theme Editor Sections group below in the same order as the Freshdesk editor.
  3. Paste `CSS`, `Head`, `Header`, `Footer`, `Layout`, and `Portal Home` into the matching Freshdesk sections.
  4. Use `theme-preview.html` for visual checking because `freshdesk-template.html` is raw Liquid and not a browser preview.

Deploy Fragment Copy/Paste

Use this when you only need the smaller rollout snippets instead of replacing full theme sections.

  1. Copy from the Deploy Fragments group below.
  2. Paste the CSS, header widget, hero block, or footer script into the corresponding admin area.
  3. Keep these snippets synced with the canonical `theme/` files rather than editing them independently.
